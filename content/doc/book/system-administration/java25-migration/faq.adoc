== FAQ

=== Who should migrate first?

Start with non-production or staging controllers, then migrate production after successful validation.

=== Do controllers and agents both need Java 25?

Yes. Jenkins remoting requires controller and agents to run compatible Java versions.

=== Should I migrate if my current Java version is stable?

Migrate when you can schedule validation and rollback windows, and when your Jenkins release line supports Java 25.

=== Can I test Java 25 before full production rollout?

Yes. Run staging tests with a production-like plugin set and representative jobs.

=== Where should Java 25 issues be reported?

Report Jenkins issues through link:/participate/report-issue/#issue-reporting[issue reporting documentation].
Include environment details, logs, and reproducible steps.

== Best practices

. Use staged rollouts with canary agents and representative workloads.
. Keep Jenkins core and plugins up to date before changing Java.
. Capture baseline metrics before migration and compare after.
. Require explicit go/no-go and rollback checkpoints.
. Document findings by operating system and CPU architecture.
. Track related Java 25 readiness work in link:https://github.com/jenkinsci/docker-agents/issues/939[jenkinsci/docker-agents#939].
