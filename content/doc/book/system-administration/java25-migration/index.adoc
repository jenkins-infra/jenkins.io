This guide helps Jenkins administrators migrate controllers and agents to Java 25 safely.

It is intended for:

* Jenkins administrators planning a Java runtime upgrade
* Plugin maintainers validating compatibility
* Contributors running early validation on Linux and macOS across x86_64 and aarch64

== Why this migration matters

Java 25 is supported for recent Jenkins weekly and LTS lines.
Moving to Java 25 keeps Jenkins installations aligned with current Java support policy and gives access to current JVM performance and diagnostics improvements.

Use this cookbook to:

* Understand compatibility expectations for Jenkins core and plugins
* Plan controller and agent migration with rollback checkpoints
* Run reproducible tests on Linux/macOS x86_64 and aarch64
* Capture issues for reporting and follow-up

== Scope and assumptions

This cookbook assumes that:

* You have an existing, working Jenkins installation
* You can stop and restart controller and agents during a maintenance window
* You can restore from backup if validation fails

For Java runtime support policy details, refer to link:/doc/book/platform-information/support-policy-java/[Java Support Policy].

For container image work related to Java 25 preview targets, refer to link:https://github.com/jenkinsci/docker-agents/issues/939[jenkinsci/docker-agents#939].
