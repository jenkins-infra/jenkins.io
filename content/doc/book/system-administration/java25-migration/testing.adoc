Use this page to run reproducible validation tests for Java 25.

== Test environments

Target environments for migration validation:

* macOS aarch64 (Apple Silicon)
* Linux aarch64
* Linux x86_64

[NOTE]
====
Use the same Jenkins core version and plugin set across test environments so results are comparable.
====

== Pre-test checklist

. Confirm Jenkins version supports Java 25.
. Update all plugins in the test environment.
. Prepare a representative job and pipeline set.
. Enable log collection for controller and agent startup.

== Example startup validation

[source,bash]
----
java -version
java --add-opens=java.base/java.lang=ALL-UNNAMED -jar jenkins.war
----

Record and classify findings:

* Illegal reflective access or module access warnings
* Deprecation notices
* Plugin initialization failures
* Agent handshake or remoting errors

== Suggested test scenarios

. Controller startup with full production-like plugin set.
. Inbound and SSH agent reconnection.
. Pipeline execution with Maven/Gradle/Node workloads.
. Job retries, workspace cleanup, and artifact archiving.
. Restart-resume behavior for long-running pipelines.

== Architecture-specific guidance

=== macOS aarch64

* Validate local developer tooling paths and shell environments.
* Verify launch method behavior for both interactive and service starts.

=== Linux aarch64

* Validate package availability and Java path consistency.
* Validate container images when agents are containerized.

=== Linux x86_64

* Treat as baseline for regression comparison.
* Compare startup time and job runtime against pre-migration baseline.

== Logging and diagnostics

Collect at minimum:

* Controller startup logs
* Agent connection logs
* Failing build logs with plugin/version context
* Java runtime details (`java -version`)

When reporting issues, include:

* Jenkins version
* Java distribution and exact build number
* Operating system and architecture
* Steps to reproduce
* Relevant stack traces
