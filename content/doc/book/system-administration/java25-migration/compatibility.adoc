Before migrating, confirm that your Jenkins version and plugin set are compatible with Java 25.

== Compatibility expectations

As documented in the Java support policy:

|===
|Jenkins line |Java 25 status

|Weekly 2.545 and newer
|Supported

|LTS 2.541.1 and newer
|Supported

|Older weekly/LTS releases
|Upgrade Jenkins first, then migrate Java
|===

[IMPORTANT]
====
Controller and agent JVM versions must match.
Plan Java 25 upgrades for both controller and all agents.
====

== Plugin compatibility notes

Most actively maintained plugins should run on Java 25 once updated to current releases.
Before production migration:

. Upgrade Jenkins to a Java 25-capable release.
. Upgrade all installed plugins.
. Review plugins with native binaries, custom classloading, or legacy reflection behavior.
. Run a staging test with representative jobs and pipelines.

Prioritize checks for:

* Authentication and security plugins
* SCM integrations
* Build tools and report publishers
* Agent launch method plugins (SSH, inbound, cloud)

== Prerequisites checklist

. Backup `JENKINS_HOME` and secrets before the first Java 25 startup.
. Export a plugin inventory from Plugin Manager.
. Confirm each agent launch method has a Java 25 runtime available.
. Confirm package or container image availability for each target OS/arch.
. Define rollback criteria and owner approval steps.
