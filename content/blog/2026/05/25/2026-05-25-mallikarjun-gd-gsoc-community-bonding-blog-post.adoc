== Project Description

The project builds on a Jenkins plugin that embeds an AI chatbot directly into the Jenkins UI, allowing users to query documentation, plugin details, and community resources without leaving Jenkins.

My work this summer extends that base with three core features:

* An *LLM-as-a-Judge evaluation pipeline* using a curated golden dataset and DeepEval metrics to score retrieval and answer quality.
* *GraphRAG via NetworkX* for plugin ecosystem queries that need to traverse dependency relationships across plugins.
* A *Build Failure Diagnosis Agent* that reads Jenkins build logs, strips PII with Presidio, and passes the sanitized context to the LLM.

== What I Did in the Community Bonding Period

I set up the local development environment on Fedora Linux — downloading the LLM model, running the data pipeline to populate the vector database, and getting the frontend and backend running together end to end.

I read through the full codebase: the RAG pipeline, plugin configuration, query flow between frontend and backend, and how responses are returned to the user.

I studied the link:https://docs.confident-ai.com/[DeepEval documentation] and the link:https://networkx.org/documentation/stable/[NetworkX] source and docs in preparation for the GraphRAG graph layer.

=== Eval Pipeline Design

I designed the evaluation pipeline architecture — dataset format, judge model selection, which metrics to track, and how to store results so that runs remain comparable over time.

=== Prototype Run

I ran a prototype pipeline on a small synthetic dataset in a personal repository before touching the main codebase.
This surfaced a real problem early: the initial judge model choice was too slow at scale, which I was able to address before it blocked any coding-period work.

== What Lies Ahead (Coding Period)

=== Golden Q&A Dataset

Build 100 curated Q&A pairs covering plugin documentation, Jenkinsfile syntax, and common build failure cases to serve as the ground truth for the evaluation pipeline.

=== DeepEval Integration

Integrate DeepEval metrics into the GitHub Actions pipeline and establish a baseline score for the current chatbot before any changes land, so improvements are measurable.

=== GraphRAG with NetworkX

Build the GraphRAG layer with NetworkX and benchmark it against flat-RAG on plugin dependency queries — validating whether graph traversal genuinely helps on these queries or just adds overhead.

=== Build Failure Diagnosis Agent

Build the agent in two stages: log ingestion first, then Presidio-based PII sanitization before passing context to the LLM.
